图层控制重复添加问题 - 调试页面

问题描述

用户反馈：点击显示控制中的图层选项时，左下角图层控制会重复添加图层选项。

修复方案

新方案：完全重建图层控制

  • 不再尝试移除单个图层
  • 每次更新时完全重建整个图层控制
  • 使用currentOverlays对象跟踪当前应该显示的图层
  • 调用rebuildLayerControl()函数重新创建图层控制

技术实现

function rebuildLayerControl() {
  if (layerControl) {
    // 移除现有的图层控制
    map.removeControl(layerControl);
    
    // 重新创建图层控制
    const baseLayers = { ... };
    const overlays = {};
    
    // 根据currentOverlays构建overlay
    if (currentOverlays["患者点图"]) {
      overlays["患者点图"] = currentOverlays["患者点图"];
    }
    if (currentOverlays["热力图"]) {
      overlays["热力图"] = currentOverlays["热力图"];
    }
    
    // 重新创建图层控制
    layerControl = L.control.layers(baseLayers, overlays, {
      position: 'bottomleft',
      collapsed: true
    }).addTo(map);
  }
}

测试步骤

  1. 打开主地图应用
  2. 多次点击"患者点图"开关（开启→关闭→开启→关闭）
  3. 多次点击"热力图"开关（开启→关闭→开启→关闭）
  4. 检查左下角图层控制中的选项数量
  5. 验证是否还会出现重复的图层选项

预期结果

修复后：无论点击多少次开关，左下角图层控制中每个图层选项都只出现一次，不会重复。

如果问题仍然存在

如果修复后问题仍然存在，可能需要考虑以下替代方案：

  • 完全禁用左下角图层控制中的overlay部分
  • 只保留底图切换功能
  • 所有图层控制都通过左上角的显示控制面板进行
